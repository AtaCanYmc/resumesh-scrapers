# Security Policy

## Supported Versions

We only support security updates for the latest stable release of `resumesh-scrapers`. Please ensure you are running the most recent version before reporting a vulnerability.

| Version | Supported |
| ------- | --------- |
| >=0.1.x | ✅ Yes     |
| <0.1.0  | ❌ No      |

## Reporting a Vulnerability

We take the security of our project seriously. If you discover a security vulnerability, please report it immediately.

**Do not open a public issue.** Publicly disclosing a vulnerability makes it easier for attackers to exploit it before a fix is available.

Instead, please report vulnerabilities by opening a private draft security advisory directly through GitHub:

1. Go to the **Security** tab of the repository on GitHub.
2. Select **Advisories** from the left-side menu.
3. Click on **New draft advisory**.
4. Fill in the details of the vulnerability and submit.

Alternatively, you can contact the maintainer directly via email at: `ata.can.ymc@gmail.com`

### Our Response Process

1. We will acknowledge receipt of your vulnerability report within 48 hours.
2. We will analyze the vulnerability to verify the issue and determine its impact.
3. We will work on a fix and test it.
4. We aim to release a patched version within 14 days of confirmation.
5. Once a fix is released, we will publish a security advisory details page to inform our users.
